## 5. Bias parity (n = 30 probes, 10 topics × 3 species)

- **Method**: for each `(topic)` group, ask the same factual question for
  three species (dog / cat / underrepresented). Measure answer length in chars.
- **Parity ratio** = `min_chars / max_chars` per group, then averaged.
- **Why this is a proxy**: if hamster answers are systematically half the
  length of dog answers, it usually signals KB sparsity for hamsters, which
  *eventually* turns into worse advice — even if the short answer happens to
  be correct.

## 8. Known limitations

> ≥ 3 honest items — what we *know* the system gets wrong, even at SoTA scores.

1. **Bias proxy is length-only.** Equal-length answers can still be unequal in
   substance. A real audit would need human raters per species.
2. **Critic is the same model family as the generator.** We get correlated
   errors — if `gpt-4o-mini` doesn't know hamster nutrition, neither does its
   critic.
3. **Knowledge base is small (~30 markdown docs).** Out-of-KB queries fall back
   on the LLM's prior, which is exactly what RAG is supposed to avoid.
